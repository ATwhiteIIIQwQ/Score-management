# 成绩管理系统

### 功能概述

#### **1. 目标用户与场景**

- **用户角色**：
  - **管理员**：负责学生信息及成绩的增删改查。
  - **学生**：仅作为被管理对象，不参与系统操作（暂不开放学生端）。
- **使用场景**：
  1. 管理员登录后，批量查看所有学生成绩；
  2. 通过学号或姓名快速定位学生，查看其单科或多科成绩；
  3. 新增学生信息或为已有学生添加/删除某一科目成绩。

#### 2.数据库设计

##### 2.1表结构设计

需设计**4张核心表**：`用户表`、`学生表`、`科目表`、`成绩表`，满足功能需求并保证数据一致性。

------

##### **表1：用户表（`user`）**

| 字段名   | 类型         | 约束                        | 说明           |
| :------- | :----------- | :-------------------------- | :------------- |
| courseId       | INT          | PRIMARY KEY, AUTO_INCREMENT | 用户ID（主键） |
| username | VARCHAR(50)  | UNIQUE, NOT NULL            | 用户名（唯一） |
| userPassword | VARCHAR(255) | NOT NULL                    |                |

**功能支持**：

- 管理员登录、密码加密、账户锁定机制。

------

##### **表2：学生表（`student`）**

| 字段名        | 类型          | 约束          | 说明        |
| :--------- | :---------- | :---------- | :-------- |
| student_id | VARCHAR(20) | PRIMARY KEY | 学号（主键，唯一） |
| studentName       | VARCHAR(50) | NOT NULL    | 学生姓名      |
| class      | VARCHAR(50) |             | 所属班级      |
| gender     | CHAR(2)     |             | 学生性别      |
| studentGrade        | INT         |             | 学生年龄      |



**功能支持**：

- 新增学生、展示学生列表、搜索学生。

------

##### **表3：科目表（`subject`）**

| 字段名       | 类型        | 约束                        | 说明                       |
| :----------- | :---------- | :-------------------------- | :------------------------- |
| subject_id   | INT         | PRIMARY KEY, AUTO_INCREMENT | 科目ID（主键）             |
| subject_name | VARCHAR(50) | UNIQUE, NOT NULL            | 科目名称（唯一，如“数学”） |
| credits      | INT         | NOT NULL                    | 学分                       |

**功能支持**：

- 科目标准化管理，避免重复录入科目名称。

------

##### **表4：成绩表（`student`）**

| 字段名     | 类型        | 约束                        | 说明                      |
| :--------- | :---------- | :-------------------------- | :------------------------ |
| courseId         | INT         | PRIMARY KEY, AUTO_INCREMENT | 成绩记录ID（主键）        |
| student_id | VARCHAR(20) | NOT NULL                    | 学号（关联`student`表）   |
| subject_id | INT         | NOT NULL                    | 科目ID（关联`subject`表） |
| student      | TINYINT     | CHECK (0≤student≤100)         | 成绩（0-100分）           |
| grade      | CHAR(1)     |                             | 学分（ABCDE）             |

**功能支持**：

- 添加/删除单科成绩、覆盖重复科目成绩、展示成绩列表。

##### 2.2表关联关系

1. **学生表（student） ↔ 成绩表（student）**

  - 一对多关系：一个学生可有多条成绩记录。
  - 外键：`student.student_id` 关联 `student.student_id`。

2. **科目表（subject） ↔ 成绩表（student）**

  - 多对多关系：一个科目可被多个学生选择，一个学生可以选择多个科目

  - 外键：`student.subject_id` 关联 `subject.subject_id`,

    ​	  `student.student_id` 关联 `student.student_id`



#### **3. 功能需求**

##### **3.1 管理员登录**

- **功能描述**：
- 管理员通过用户名和密码登录系统。
- **交互逻辑**：

  1. 输入用户名、密码；
2. 验证通过后跳转到主页面；

##### 3.2 管理员注册

* 功能描述：

  * 管理员通过个人信息进行注册

* 交互逻辑：

  1.输入id，用户名和密码；

  2.检查id是否重复；

  3.id重复则提示用户重新输入id

  4.注册成功后跳转回登录页

##### **3.2 全部学生成绩展示**

- **功能描述**：

  - 以列表形式展示所有学生的姓名、学号、各科成绩及总分。

- **交互逻辑**：

  1. 默认按学号升序排列；
  2. 支持按总分/单科成绩排序；
  3. 支持分页显示（每页20条）。



##### **3.3 搜索单个学生成绩**

- **功能描述**：

  - 通过学号或姓名搜索学生，显示其所有科目成绩。

- **交互逻辑**：

  1. 在搜索框输入学号或姓名（支持模糊查询）；
  2. 显示匹配学生的详细信息及成绩；
  3. 若无匹配结果，提示“未找到相关学生”。



##### **3.4 新增学生**

- **功能描述**：

  - 添加一名新学生的基本信息（学号、姓名、班级）。

- **交互逻辑**：

  1. 填写学号（必填、唯一）、姓名（必填）、班级（可选）；
  2. 提交后校验学号是否重复；
  3. 成功提示“添加成功”，失败提示具体原因。



##### **3.5 为指定学生添加成绩**

- **功能描述**：
  - 为已有学生录入某一科目的成绩。
- **交互逻辑**：
  1. 搜索目标学生；
  2. 选择科目（如数学、语文），输入成绩（0-100）；
  3. 提交后更新学生成绩列表。
- **验收标准**：
  - 同一学生同一科目仅允许一条记录（重复提交则覆盖旧数据）。

##### **3.6 删除指定学生的单科成绩**

- **功能描述**：

  - 删除某学生的某一科目成绩记录。

- **交互逻辑**：

  1. 搜索目标学生；
  2. 在成绩列表中找到对应科目，点击“删除”；
  3. 弹出确认对话框，确认后删除数据。
